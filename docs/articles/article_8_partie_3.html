Saccadic-JEPA — Partie 3 : Entraînement SSL Haute Résolution

Apprendre des structures nettes à partir d'indices flous

Série Saccadic-JEPA : Vers une Vision Artificielle Biologiquement Inspirée Dans cette troisième et dernière partie de notre huitième article, nous nous attaquons à la sécurité de l'apprentissage. Les réseaux de neurones sont des tricheurs nés : s'il existe un raccourci pour minimiser la perte sans comprendre l'image, ils le trouveront. Découvrons comment le masquage strict et asymétrique empêche l'Élève de copier sur le Professeur.

🦊 La loi du moindre effort en Machine Learning

En apprentissage supervisé comme en auto-supervisé, les modèles souffrent d'un biais fondamental : l'apprentissage par raccourcis (shortcut learning). Si un réseau peut résoudre un problème en mémorisant un bruit de fond, un pixel spécifique, ou en faisant un simple copier-coller de texture, il le fera, au détriment de l'apprentissage de véritables concepts sémantiques.

Dans Saccadic-JEPA, nous voulons que le Predictor devine la structure nette d'une zone cible $B$ à partir d'un contexte flouté centré sur $A$.

Mais que se passerait-il si l'encodeur de contexte (l'Élève) pouvait voir l'emplacement de la zone cible $B$ pendant qu'il analyse le contexte ? 1. Si la cible $B$ est proche du regard $A$ : Elle apparaîtra nette dans l'image fovéée. L'Élève encodera directement les détails fins de la cible. Le Predictor n'aura plus qu'à copier ces informations. L'apprentissage est saboté. 2. Si la cible $B$ est loin du regard $A$ : Elle apparaîtra floue. L'Élève encodera la forme grossière de la cible et la transmettra au Predictor. Ce dernier n'aura qu'à faire un simple travail de "défloutage" local de caractéristiques, sans avoir besoin de déduire la cible à partir de son environnement.

🛡️ La parade : L'exclusion stricte par masquage d'index

Pour empêcher ce sabotage, Saccadic-JEPA combine la fovéation optique et le masquage par index disjoints.

Le mécanisme repose sur le découpage en patches du Vision Transformer. Sur notre image de $128\times 128$ pixels découpée en patches de $16\times 16$, nous obtenons une grille de $8\times 8 = 64$ patches.

                  GRILLE DE 64 PATCHES (8x8)

         ┌───┬───┬───┬───┬───┬───┬───┬───┐
         │ C │ C │ C │ C │ C │ C │ C │ C │   C = Patches du Contexte
         ├───┼───┼───┼───┼───┼───┼───┼───┤       (Visibles par l'élève,
         │ C │ C │ C │ C │ C │ C │ C │ C │        flous en périphérie de A)
         ├───┼───┼───┼───┼───┼───┼───┼───┤
         │ C │ C │ T │ T │ T │ T │ C │ C │   T = Patches de la Cible
         ├───┼───┼───┼───┼───┼───┼───┼───┤       (100% invisibles pour l'élève,
         │ C │ C │ T │ T │ T │ T │ C │ C │        extraits nets par le prof)
         ├───┼───┼───┼───┼───┼───┼───┼───┤
         │ C │ C │ T │ T │ T │ T │ C │ C │
         ├───┼───┼───┼───┼───┼───┼───┼───┤
         │ C │ C │ T │ T │ T │ T │ C │ C │
         ├───┼───┼───┼───┼───┼───┼───┼───┤
         │ C │ C │ C │ C │ C │ C │ C │ C │
         └───┴───┴───┴───┴───┴───┴───┴───┘

Lors de chaque passe d'entraînement : 1. La fonction sample_masks sélectionne un grand bloc cible $T$ de $4\times 4$ patches. 2. Elle sépare la liste des 64 index de patches en deux sous-ensembles : * target_indices : Les 16 patches appartenant au bloc cible. * context_indices : Les 48 patches restants. 3. Règle d'or : L'intersection entre ces deux groupes est strictement vide : $$\text{Context Indices} \cap \text{Target Indices} = \emptyset$$

Lorsque nous alimentons le Context Encoder (self.context_encoder), nous ne lui donnons que la liste des context_indices. Le réseau ne reçoit tout simplement pas les tokens de la cible. Pour lui, la zone cible n'existe pas, elle est physiquement absente de sa mémoire de calcul.

💻 Zoom sur le code de sélection disjoint : jepa_lab/masks.py

C'est au cœur de la fonction sample_masks que ce mur de sécurité est érigé :

# Sélection aléatoire de l'origine du bloc cible (carré de taille target_size)
row = random.randint(0, grid_size - target_size)
col = random.randint(0, grid_size - target_size)

# Récupération de tous les index appartenant à ce carré
target_patches = []
for r in range(row, row + target_size):
    for c in range(col, col + target_size):
        target_patches.append(r * grid_size + c)

# Le contexte est défini par complémentarité absolue (anti-triche)
all_patches = set(range(grid_size * grid_size))
context_patches = list(all_patches - set(target_patches))

Grâce à cette soustraction d'ensembles (all_patches - set(target_patches)), la triche est mathématiquement impossible.

👁️ L'effet combiné : Fovéation + Masquage

Dans Saccadic-JEPA, ces deux forces s'associent pour créer la tâche idéale d'apprentissage auto-supervisé : * Le regard du modèle est fixé sur le point $A$ (la fovéa du contexte est nette, la périphérie est floue). * La cible à deviner se trouve sur le bloc $B$. * Le bloc $B$ est découpé et jeté à la poubelle pour l'Élève. * Le Predictor doit reconstruire la signature sémantique nette de $B$ en s'appuyant uniquement sur les patches contextuels restants, qui sont de plus en plus flous à mesure qu'ils s'éloignent de $A$.

C'est cette double contrainte — ne pas voir la cible et n'avoir qu'un contexte dégradé en périphérie — qui force le modèle à développer des représentations latentes d'une robustesse exceptionnelle, capables de capturer l'essence géométrique globale des images.

🧠 Conclusion

Vous comprenez désormais l'architecture complète de Saccadic-JEPA : une simulation d'œil fovéé, un encodeur élève plongé dans un contexte asymétrique et masqué, un prof détenant la vérité haute résolution, et un Predictor jouant le rôle de traducteur géométrique.

Dans le prochain et dernier article, nous allons analyser les résultats empiriques de ce modèle sur CIFAR-10 et comprendre pourquoi la route vers la performance passe par un entraînement à l'échelle de 120 époques.
