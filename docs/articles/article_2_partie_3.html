Nous verrons comment le choix géométrique de la zone masquée détermine si le réseau va apprendre la sémantique des objets ou simplement tricher en faisant du coloriage local.

🎨 Le coloriage vs l'imagination sémantique

Pour comprendre l'importance de la géométrie du masquage, faisons une expérience humaine.

Prenez un dessin de chat.

  • Cas A (Masquage aléatoire) : On efface un millier de petits points de 1 pixel répartis partout sur le dessin (comme du bruit "sel et poivre"). On vous demande de compléter le dessin. C'est très facile : il vous suffit de relier les lignes et de prolonger les couleurs locales. Vous n'avez pas besoin de savoir que c'est un "chat" pour réussir.
  • Cas B (Masquage par blocs) : On pose un grand carton blanc rectangulaire en plein milieu de l'image, cachant toute la tête du chat. On vous demande ce qu'il y a sous le carton. Pour deviner qu'il y a des moustaches, des yeux et des oreilles pointues, vous devez analyser le reste de l'image (les pattes, la queue, le pelage) et mobiliser votre concept mental d'un chat.

🚫 Le piège de l'interpolation basse fréquence

Dans les architectures comme les Masked Autoencoders (MAE), on masque généralement de 75% à 80% des patches de manière totalement aléatoire. C'est une stratégie redoutable pour les très grands modèles de pixels, mais elle présente un défaut pour l'apprentissage de représentations pures : elle encourage l'interpolation locale.

Le réseau apprend à agir comme un filtre de lissage (un algorithme d'inpainting). Il se focalise sur les hautes fréquences (les bords de lignes, le gradient de couleur) pour "boucher les trous" adjacents.

La philosophie JEPA (Meta, Yann LeCun) prend le contre-pied : En masquant un unique grand bloc contigu (par exemple, un carré de $4\times 4$ patches représentant 25% de notre image CIFAR-10), on s'assure que :

  1. Les patches situés au centre de la cible n'ont aucun voisin visible à proximité immédiate.
  2. Le modèle ne peut pas tricher par simple interpolation locale.
  3. Pour prédire l'embedding du centre de la cible, le Predictor doit s'appuyer sur la structure globale de l'image capturée par le Context Encoder.

🛠️ Dans les entrailles du code : jepa_lab/masks.py

Voyons comment cette sélection géométrique s'implémente en PyTorch dans notre fichier masks.py. La fonction centrale est sample_masks : 

def sample_masks(batch_size, grid_size=8, target_size=4, device='cpu'):
    # grid_size = 8 (car grille 8x8 patches sur CIFAR-10)
    # target_size = 4 (car cible de 4x4 patches)

    context_idx_list = []
    target_idx_list = []

    for _ in range(batch_size):
        # 1. Choisir aléatoirement le coin supérieur gauche du bloc cible
        r = random.randint(0, grid_size - target_size) # Index de ligne (0 à 4)
        c = random.randint(0, grid_size - target_size) # Index de colonne (0 à 4)

        # 2. Collecter les index de tous les patches appartenant à ce bloc
        target_patches = []
        for i in range(target_size):
            for j in range(target_size):
                target_patches.append((r + i) * grid_size + (c + j))

        # 3. Le contexte est le complément strict (tous les autres patches)
        all_patches = list(range(grid_size * grid_size))
        context_patches = [p for p in all_patches if p not in target_patches]

        context_idx_list.append(torch.tensor(context_patches, dtype=torch.long, device=device))
        target_idx_list.append(torch.tensor(target_patches, dtype=torch.long, device=device))

    return torch.stack(context_idx_list), torch.stack(target_idx_list)

🔍 Décryptage de l'algorithme :

  1. Ajustement des limites : random.randint(0, grid_size - target_size) garantit que le bloc cible ne dépasse jamais de l'image (il reste confiné dans la grille $8\times 8$).
  2. Complémentarité mathématique : [p for p in all_patches if p not in target_patches] crée une cloison étanche. Aucun patch cible ne se retrouve dans le contexte. Le Context Encoder a l'interdiction absolue de voir la moindre information sur la cible.

🎬 Bilan de l'Article 2

Nous avons maintenant toutes les clés pour comprendre notre premier JEPA mono-cible :

  1. L'image est découpée en 64 patches projetés dans un espace de dimension 192, enrichis de coordonnées spatiales (Partie 1).
  2. Un jeu à trois acteurs s'installe : le Context Encoder encode le visible, le Target Encoder produit la cible stable sous contrôle de l'EMA (sans gradients), et le Predictor spécule dans l'espace latent (Partie 2).
  3. Le masque par grand bloc de $4\times 4$ force le modèle à mobiliser des concepts globaux plutôt qu'à faire du simple coloriage local (Partie 3).

    Notre prototype mono-cible fonctionne et s'entraîne sur CIFAR-10. Mais comment passer à la vitesse supérieure et implémenter l'architecture multi-cibles parallèle décrite dans le papier de recherche de Meta ?

En revanche, si vous masquez un paragraphe entier de trois lignes contiguës, le lecteur doit comprendre tout le contexte du document pour déduire l'idée générale censurée.
