Cartographie Locale & Algorithme BFS — Partie 4 : La Projection des BBoxes

Localiser plusieurs objets à la fois

Série Recherche Visuelle & Localisation d'Objets sans Supervision Dans cette quatrième partie de notre onzième article, nous découvrons la dernière étape clé de notre pipeline de recherche : la projection géométrique. Les objets détectés par notre algorithme BFS sont représentés par des listes de patches à basse résolution (dans une grille d'activation). Comment transposer ces coordonnées discrètes de patches pour tracer des boîtes englobantes (BBoxes) extrêmement précises sur l'image d'origine haute résolution ? Nous présentons les calculs mathématiques et la logique de code sous-jacente.

🌉 Étape 1 : Le pont entre deux résolutions

Pour comprendre le besoin de projection, récapitulons la chaîne de traitement d'une image de test : 1. L'image d'origine a des dimensions arbitraires, par exemple $1920\times1080$ pixels ($W_{orig} \times H_{orig}$). 2. L'image d'entrée du ViT est redimensionnée à une taille fixe requise par le modèle (par exemple, $128\times128$ pixels pour Saccadic-JEPA). C'est la taille img_size. 3. La grille de similarité est quantifiée par la taille des patches (par exemple, des patches de $16\times16$ pixels, soit une grille $8\times8$).

L'algorithme BFS de la partie précédente nous renvoie une composante connexe, c'est-à-dire une liste de coordonnées sur la grille $8\times8$, par exemple : comp = [(3, 2), (3, 3)].

Nous devons convertir ces coordonnées de grille en coordonnées de pixels réels sur l'image $1920\times1080$ afin de pouvoir y dessiner notre cadre de détection.

🧮 Étape 2 : Les formules de changement de repère

Le passage des indices de la grille aux pixels de l'image finale s'effectue en deux étapes de mise à l'échelle.

1. Passage de la grille à l'image redimensionnée (resized)

On commence par trouver les limites de notre composante connexe dans la grille (les lignes minimale $r_{min}$ et maximale $r_{max}$, et les colonnes minimale $c_{min}$ et maximale $c_{max}$) :

$$r_{min} = \min(\{r\}), \quad r_{max} = \max(\{r\})$$ $$c_{min} = \min(\{c\}), \quad c_{max} = \max(\{c\})$$

Chaque patch mesurant patch_size pixels dans l'image redimensionnée de $128\times128$, nous convertissons les indices en coordonnées de pixels :

$$y_{min\_resized} = r_{min} \times \text{patch\_size}$$ $$x_{min\_resized} = c_{min} \times \text{patch\_size}$$ $$y_{max\_resized} = (r_{max} + 1) \times \text{patch\_size}$$ $$x_{max\_resized} = (c_{max} + 1) \times \text{patch\_size}$$

Important

Pourquoi le $+1$ ? Le patch d'indice maximum $r_{max}$ s'étend de la coordonnée $r_{max} \times \text{patch\_size}$ jusqu'au bord opposé du patch. Si on omettait le $+1$, la boîte englobante oublierait d'inclure la largeur et la hauteur du dernier patch, amputant notre détection d'une partie de l'objet.

2. Passage de l'image redimensionnée à l'image d'origine

On applique ensuite un ratio linéaire simple pour projeter ces coordonnées sur l'image d'origine ($W_{orig} \times H_{orig}$), tout en veillant à ne pas dépasser les limites physiques de l'image (écrêtage entre 0 et la dimension maximale) :

$$x_{min} = \text{int}\left( \max\left(0, \frac{x_{min\_resized}}{\text{img\_size}} \times W_{orig}\right) \right)$$ $$x_{max} = \text{int}\left( \min\left(W_{orig}, \frac{x_{max\_resized}}{\text{img\_size}} \times W_{orig}\right) \right)$$ $$y_{min} = \text{int}\left( \max\left(0, \frac{y_{min\_resized}}{\text{img\_size}} \times H_{orig}\right) \right)$$ $$y_{max} = \text{int}\left( \min\left(H_{orig}, \frac{y_{max\_resized}}{\text{img\_size}} \times H_{orig}\right) \right)$$

🛠️ Dans les entrailles du code : La fonction de traitement

Voici comment cette logique de projection géométrique est programmée dans notre fonction process_target_image du fichier unsupervised_search.py : 

# ... Extrait du traitement des composantes connexes ...
detections = []
for comp in components:
    # Ignorer les grappes trop petites pour filtrer le bruit résiduel
    if len(comp) < min_patches:
        continue

    # 1. Bornes de la composante dans l'espace grille (8x8)
    r_coords = [r for r, c in comp]
    c_coords = [c for r, c in comp]

    r_min, r_max = min(r_coords), max(r_coords)
    c_min, c_max = min(c_coords), max(c_coords)

    # 2. Conversion vers l'espace de l'image redimensionnée (128x128)
    y_min_resized = r_min * patch_size
    y_max_resized = (r_max + 1) * patch_size
    x_min_resized = c_min * patch_size
    x_max_resized = (c_max + 1) * patch_size

    # 3. Projection homothétique vers l'image d'origine (ex: 1920x1080)
    y_min = int(max(0, (y_min_resized / img_size) * H_orig))
    y_max = int(min(H_orig, (y_max_resized / img_size) * H_orig))
    x_min = int(max(0, (x_min_resized / img_size) * W_orig))
    x_max = int(min(W_orig, (x_max_resized / img_size) * W_orig))

    # 4. Calcul de l'indice de confiance (similarité moyenne des patches)
    comp_scores = [sim_grid[r, c] for r, c in comp]
    mean_score = np.mean(comp_scores)
    max_score = np.max(comp_scores)

    detections.append({
        'bbox': [x_min, y_min, x_max, y_max],
        'mean_score': float(mean_score),
        'max_score': float(max_score),
        'patch_count': len(comp)
    })

📈 Étape 3 : Confiance et tri des détections

En plus de projeter la boîte englobante, le script calcule un score de confiance pour chaque objet détecté :

  • Score moyen (mean_score) : C'est la moyenne arithmétique des similarités cosinus de tous les patches formant l'objet. Ce score indique la fidélité globale de la région détectée vis-à-vis du concept recherché.
  • Score maximal (max_score) : Le score individuel le plus élevé au sein de la grappe, signalant le cœur le plus proche du concept.

Toutes les détections sont ensuite triées par score moyen décroissant. Cela permet à l'utilisateur de n'afficher ou de ne conserver que les occurrences les plus fiables dans l'image.

🧠 Conclusion de la Partie 2

Nous avons à présent complété toute l'ingénierie algorithmique de notre système de recherche visuelle sans supervision !

  1. Nous avons compris comment une image est découpée, et comment la similarité cosinus de l'espace latent remplace la comparaison naïve de pixels (Article 10).
