De la Similarité Pixel à la Sémantique Latente — Partie 2 : Le Vecteur de Concept (q)

Comment l'IA comprend les objets sans labels

Série Recherche Visuelle & Localisation d'Objets sans Supervision Dans cette deuxième partie, nous expliquons comment un Vision Transformer (ViT) pré-entraîné avec la méthode JEPA convertit une simple image recadrée (crop de requête) en une représentation mathématique universelle de l'objet : le Vecteur de Concept $q$. Nous verrons comment le mécanisme de Self-Attention propage le contexte entre les patches, et comment une simple moyenne globale permet d'extraire la substantifique moelle d'un objet en filtrant le bruit haute fréquence.

✂️ Étape 1 : Le "Crop" de Requête

Pour rechercher un objet dans une base de données sans étiquettes (par exemple, localiser toutes les roues de voitures dans une série d'images de rue), la première étape consiste à fournir un exemple visuel à l'algorithme. C'est le crop de requête : une sous-image découpée contenant l'objet d'intérêt.

Cette sous-image de taille variable est d'abord redimensionnée aux dimensions attendues par notre Vision Transformer (ViT). Comme toute image ingérée par un ViT, elle est ensuite découpée en une grille de petits carrés appelés patches (par exemple, des patches de $16\times16$ pixels).

       Crop Requête (ex: Roue)            Découpage en Patches (Tokens)
       ┌─────────────────────┐               ┌─────┬─────┬─────┬─────┐
       │         ███         │               │     │  █  │  █  │     │  -> t_1
       │       ██   ██       │               ├─────┼─────┼─────┼─────┤
       │       ██   ██       │     ───>      │  █  │     │     │  █  │  -> t_2
       │         ███         │               ├─────┼─────┼─────┼─────┤
       └─────────────────────┘               │  █  │     │     │  █  │  -> ...
                                             ├─────┼─────┼─────┼─────┤
                                             │     │  █  │  █  │     │  -> t_N
                                             └─────┴─────┴─────┴─────┘

Chaque patch est projeté linéairement pour former un vecteur de départ, appelé token (jeton de patch). À ce stade, chaque jeton ne contient que des informations très locales et isolées (par exemple, "un morceau de courbe noire"). Il n'a aucune idée de l'image globale.

🕸️ Étape 2 : Le mécanisme de Self-Attention : Le tisseur de contexte

C'est ici qu'intervient la magie du Vision Transformer (ViT) pré-entraîné par JEPA. Contrairement aux réseaux de neurones convolutifs (CNN) où l'information ne se propage que très localement couche après couche, le mécanisme de Self-Attention (Auto-Attention) du ViT permet à tous les patches de communiquer directement entre eux, dès la première couche.

Mathématiquement, pour chaque patch, l'attention calcule une relation entre sa requête ($Q$), les clés de tous les autres patches ($K$), et pondère leurs valeurs ($V$) :

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

💡 L'analogie du Puzzle

Imaginez que vous essayiez de reconstituer un puzzle de 100 pièces.

  • Un modèle classique regarde chaque pièce dans son coin, essayant de deviner ce qu'elle représente à partir de sa seule texture locale.
  • Le mécanisme de Self-Attention est comparable à une réunion où toutes les pièces discuteraient ensemble en même temps. La pièce $A$ ("caoutchouc noir courbé") demande à la pièce $B$ ("jante en métal argenté") : "Es-tu située à ma droite ?". $B$ répond positivement. Ensemble, elles communiquent avec les autres pièces courbées et en déduisent : "Nous formons une structure circulaire fermée. Nous sommes une roue."

Grâce à ce tissage de relations, après quelques couches d'attention, chaque vecteur de patch est contextualisé. Le vecteur du patch contenant "du caoutchouc noir" a intégré la présence des patches voisins de "jante en métal" et "d'asphalte". Il ne code plus un simple motif de couleur, mais sa participation au concept global de "roue".

📊 Étape 3 : Le Vecteur de Concept ($q$) comme Moyenne Globale

Une fois que l'image de requête a traversé l'encodeur ViT (model.context_encoder), nous obtenons une liste de $N$ vecteurs contextualisés (un par patch) :

$$Z = \{z_1, z_2, \dots, z_N\} \quad \text{où } z_i \in \mathbb{R}^D$$

Chaque $z_i$ exprime la sémantique de l'objet vue depuis la perspective du patch $i$. Pour obtenir une représentation mathématique unique et stable de l'ensemble de l'objet, on effectue un Global Average Pooling (une moyenne arithmétique simple de tous les jetons de patches) :

$$q = \frac{1}{N} \sum_{i=1}^N z_i$$

Ce vecteur résultant $q \in \mathbb{R}^D$ est notre Vecteur de Concept.

🛡️ Pourquoi la moyenne filtre-t-elle le bruit haute fréquence ?

Pourquoi faire une simple moyenne fonctionne-t-il si bien ? En traitement du signal, faire la moyenne de plusieurs mesures permet de réduire le bruit aléatoire. Il en va de même dans l'espace sémantique latent :

  1. Annulation du bruit haute fréquence : Les détails très spécifiques de texture locale (le grain du pneu, une poussière sur la jante, un reflet lumineux parasite sur un pixel précis) varient de manière chaotique d'un patch à l'autre. Lors de la sommation, ces variations locales (haute fréquence) s'annulent mutuellement.
  2. Amplification du signal sémantique (basse fréquence) : Le point commun partagé par tous ces patches est leur appartenance à la structure globale de l'objet. Ce signal sémantique persistant (la structure géométrique circulaire, le contraste roue/sol) est renforcé par la moyenne.

Le vecteur $q$ qui en résulte capture ainsi la structure abstraite et robuste de l'objet, le rendant insensible aux micro-variations.

🛠️ Dans les entrailles du code : L'extraction de $q$

Voici comment cette extraction se traduit concrètement en PyTorch dans notre pipeline de recherche d'objets :

import torch
import torch.nn.functional as F

def extract_concept_vector(model, crop_image):
    """
    Extrait le vecteur de concept q à partir d'une image de requête (crop).

    Arguments:
        model: Le modèle JEPA pré-entraîné (contenant le context_encoder)
        crop_image: Tenseur PyTorch de la requête de forme (1, 3, H, W)
    """
    model.eval()
    with torch.no_grad():
        # 1. Passage dans l'encodeur de contexte du ViT
        # Renvoie les embeddings de patches : (1, N_patches, D)
        patch_embeddings = model.context_encoder(crop_image)

        # 2. Moyenne globale sur la dimension des patches (dim=1)
        # Renvoie un vecteur unique : (1, D)
        q = patch_embeddings.mean(dim=1)

        # 3. Normalisation L2 (essentielle pour la similarité cosinus)
        q = F.normalize(q, p=2, dim=-1)

    return q

🧠 Conclusion & Transition

En combinant le mécanisme de Self-Attention (qui contextualise les patches) et la moyenne globale (qui élimine les détails superflus), nous avons obtenu un vecteur unique $q$ qui encapsule proprement le concept de l'objet recherché.

Mais une fois ce vecteur de concept $q$ en main, comment l'utiliser pour scanner une nouvelle image test et y repérer nos objets ? Comment mesure-t-on la proximité entre $q$ et les patches de l'image cible ?
