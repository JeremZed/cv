Fovéation, cônes/bâtonnets et acuité visuelle

Série Saccadic-JEPA : Vers une Vision Artificielle Biologiquement Inspirée Dans cette troisième et dernière partie du septième article, nous passons à la pratique. Nous allons voir comment modéliser mathématiquement le gradient de netteté de l'œil sans effondrer les performances de calcul, et comment exécuter notre propre simulateur de fovéation en Python.

🐢 Le piège de l'approche naïve

Si vous essayez de coder un flou fovéal de manière intuitive, vous risquez de vous heurter à un mur de lenteur.

L'approche naïve consisterait à dire : "Pour chaque pixel de l'image, je calcule sa distance au point de regard, j'en déduis un rayon de flou gaussien spécifique (petit au centre, grand en périphérie), et j'applique un filtre local."

Bien que correcte sur le plan théorique, cette méthode est un désastre informatique. Faire varier la taille du noyau de convolution pour chaque pixel nécessite des boucles imbriquées complexes. Traiter une seule image de taille moyenne prendrait plusieurs secondes en Python. Dans un pipeline d'apprentissage où notre GPU doit ingérer des centaines d'images par seconde, cette approche est inutilisable.

⚡ La solution : Le mélange de calques multi-résolutions

Pour contourner cette limite, nous utilisons une technique empruntée au jeu vidéo et au traitement d'images professionnel : le multi-resolution layer blending.

Le principe est simple : 1. Génération de calques globaux : On crée une poignée de versions uniformément floutées et désaturées de l'image d'origine (par exemple 4 calques : Nette, Flou Léger, Flou Moyen, Flou Fort). 2. Matrice des distances : On calcule une seule fois la distance de chaque pixel au centre du regard $(c_x, c_y)$ sur toute l'image (opération matricielle ultra-rapide). 3. Fusion par masques de transition : On mélange ces calques en utilisant des masques de transparence douce pour obtenir un rendu dégradé parfait.

Cette suite d'opérations est entièrement vectorisée en NumPy (ou en PyTorch sur GPU), ce qui la rend quasi-instantanée.

📐 Les mathématiques des masques sigmoïdes

Pour obtenir des transitions fluides sans voir de "cercles" ou de "marches d'escalier" artificiels sur l'image fovéée, nous utilisons la fonction logistique (ou sigmoïde).

La fonction sigmoïde permet de définir des transitions douces entre 0 (transparence totale) et 1 (opacité totale) autour d'un rayon de transition $r$ avec une pente $s$ :

$$\text{sigmoid}(d, r, s) = \frac{1}{1 + e^{-s \cdot (d - r)}}$$

Pour modéliser nos 4 zones de vision (Fovéa, Para-fovéa, Péri-fovéa, Périphérie), nous définissons trois rayons de démarcation ($r_0, r_1, r_2$) et calculons trois masques sigmoïdes ($P_0, P_1, P_2$).

Nous calculons ensuite les poids de mélange ($W_0, W_1, W_2, W_3$) de chaque calque pour chaque pixel : * Poids du calque Net (Fovéa) : $$W_0 = 1.0 - P_0$$ * Poids du calque Flou Léger (Para-fovéa) : $$W_1 = P_0 \times (1.0 - P_1)$$ * Poids du calque Flou Moyen (Péri-fovéa) : $$W_2 = P_0 \times P_1 \times (1.0 - P_2)$$ * Poids du calque Flou Fort (Périphérie) : $$W_3 = P_0 \times P_1 \times P_2$$

💡 Propriété mathématique cruciale : La somme de ces poids est égale à exactement $1.0$ en tout point de l'image ($W_0 + W_1 + W_2 + W_3 = 1.0$), ce qui garantit qu'aucune perte ou gain d'intensité lumineuse n'apparaît lors de la fusion.

🛠️ Dans les entrailles du code : jepa_lab/explications/foveate_image.py

Le script foveate_image.py applique cette logique en Python. Voyons comment la grille de distances et la fusion sont codées :

# 1. Grille de coordonnées et distances radiales
x = np.arange(W)
y = np.arange(H)
xx, yy = np.meshgrid(x, y)
dist = np.sqrt((xx - cx)**2 + (yy - cy)**2)
dist_norm = dist / max_dist  # Distance normalisée [0, 1]

# 2. Calcul des probabilités de transition (sigmoïdes)
P0 = 1 / (1 + np.exp(-slope0 * (dist_norm - r0)))
P1 = 1 / (1 + np.exp(-slope1 * (dist_norm - r1)))
P2 = 1 / (1 + np.exp(-slope2 * (dist_norm - r2)))

# 3. Calcul des poids de calques
W0 = 1.0 - P0             # Fovéa
W1 = P0 * (1.0 - P1)      # Para-fovéa
W2 = P0 * P1 * (1.0 - P2) # Péri-fovéa
W3 = P0 * P1 * P2         # Périphérie

# 4. Fusion des calques (élargie sur la dimension couleur RGB)
W0 = np.expand_dims(W0, axis=-1)
W1 = np.expand_dims(W1, axis=-1)
W2 = np.expand_dims(W2, axis=-1)
W3 = np.expand_dims(W3, axis=-1)

# Image fovéée finale
foveated = W0 * layer_fovea + W1 * layer_para + W2 * layer_peri + W3 * layer_peripheral

🚀 Comment lancer le simulateur ?

Vous pouvez tester le script directement sur votre machine pour générer des visuels.

1. Générer une image de test synthétique

Pour évaluer la perte de lisibilité des textes en vision périphérique, générez une image contenant des mires de texte :

python3 jepa_lab/explications/foveate_image.py --output vision_humaine_clean.png

2. Afficher les limites des zones (Mode Débug)

Dessinez les cercles correspondants aux rayons de transition ($r_0, r_1, r_2$) pour visualiser les limites rétiniennes :

python3 jepa_lab/explications/foveate_image.py --debug --output vision_humaine_debug.png

3. Appliquer l'effet sur votre propre image

Vous pouvez cibler un point de regard précis sur une de vos images (ex: X=450, Y=300) :

python3 jepa_lab/explications/foveate_image.py --image votre_image.jpg --x 450 --y 300 --output vision_perso.png
