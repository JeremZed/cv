De la Similarité Pixel à la Sémantique Latente — Partie 1 : Le piège du pixel-à-pixel

Comment l'IA comprend les objets sans labels

Série Recherche Visuelle & Localisation d'Objets sans Supervision Dans cette première partie, nous explorons pourquoi les approches traditionnelles de comparaison d'images au niveau des pixels échouent systématiquement face à la recherche d'objets en conditions réelles. Nous verrons comment le bruit de fond, les variations de luminosité ou les rotations trompent les métriques pixel-à-pixel, ouvrant la voie à la nécessité d'un espace latent sémantique.

👁️ L'illusion de la vision machine : Une grille de chiffres

Pour un être humain, identifier un objet dans une image est un processus sémantique immédiat. Si l'on vous montre une pomme rouge, que celle-ci soit posée sur une table en bois clair, photographiée en plein soleil, plongée dans l'ombre d'un verger, ou inclinée à 45 degrés, votre cerveau n'hésite pas une seconde : c'est une pomme.

Pour un ordinateur, en revanche, une image n'est rien d'autre qu'une immense grille bidimensionnelle de nombres (des intensités de pixels allant de 0 à 255 pour chaque canal de couleur Rouge, Vert, Bleu).

Si l'on cherche à concevoir un système de recherche d'objets naïf (sans supervision) en demandant simplement à la machine de comparer les valeurs de pixels d'une image de requête (notre pomme) à celles d'une image de recherche, on tombe immédiatement dans un piège mathématique redoutable.

📏 La comparaison pixel-à-pixel : Une approche naïve

La méthode la plus directe pour comparer deux images $I_1$ et $I_2$ de mêmes dimensions consiste à mesurer leur distance mathématique pixel par pixel. On utilise généralement la distance euclidienne (L2) ou l'erreur quadratique moyenne (MSE) :

$$L_2(I_1, I_2) = \sqrt{\sum_{i,j} (I_1[i,j] - I_2[i,j])^2}$$

Pour comprendre pourquoi cette métrique est aveugle à la sémantique des objets, prenons un exemple ultra-simplifié avec une image en noir et blanc de dimensions $3\times3$.

Imaginons une image Image A représentant une ligne verticale blanche sur fond noir :

[  0, 255,   0 ]
[  0, 255,   0 ]
[  0, 255,   0 ]

Créons maintenant une Image B où la ligne verticale est simplement décalée d'un pixel vers la droite :

[  0,   0, 255 ]
[  0,   0, 255 ]
[  0,   0, 255 ]

Pour notre œil, il s'agit exactement du même objet (la ligne blanche), ayant subi une légère translation. Calculons la différence absolue pixel à pixel $|Image\ A - Image\ B|$ :

[  0, 255, 255 ]
[  0, 255, 255 ]
[  0, 255, 255 ]

La somme des différences au carré est de : $$(255)^2 \times 6 = 65\,025 \times 6 = 390\,150$$

Comparons maintenant notre Image A avec une image totalement noire, l'Image C :

[  0,   0,   0 ]
[  0,   0,   0 ]
[  0,   0,   0 ]

La différence absolue pixel à pixel $|Image\ A - Image\ C|$ est :

[  0, 255,   0 ]
[  0, 255,   0 ]
[  0, 255,   0 ]

La somme des différences au carré vaut ici : $$(255)^2 \times 3 = 65\,025 \times 3 = 195\,075$$

Attention

Le verdict absurde des pixels : La distance mathématique entre la ligne originale et sa version décalée ($390\,150$) est deux fois plus grande que la distance entre la ligne et le vide absolu ($195\,075$). Pour un algorithme pixel-à-pixel, un objet légèrement déplacé ressemble plus à du néant qu'à lui-même ! C'est ce qu'on appelle le piège de l'alignement spatial.

🌪️ Les trois grands pièges de l'espace pixel

Dans le monde réel, ce problème d'alignement est amplifié par trois facteurs majeurs qui rendent la comparaison de pixels inutilisable pour la recherche d'objets.

🌳 1. Le bruit de fond et l'arrière-plan dominant

Lorsqu'on effectue une recherche d'objet, l'objet ciblé (par exemple, un oiseau) ne représente souvent qu'une petite portion de l'image (disons 15% des pixels). Le reste est constitué de l'arrière-plan (du feuillage, du ciel, des nuages).

  • Si vous comparez un oiseau sur fond de feuillage vert avec le même oiseau volant dans un ciel bleu, les 85% de pixels d'arrière-plan vont dominer massivement le calcul de distance.
  • La métrique pixel conclura que l'oiseau sur fond vert ressemble plus à une pelouse vide qu'à l'oiseau sur fond bleu.

☀️ 2. Les variations de luminosité et d'éclairage

La lumière change constamment. Si un nuage passe devant le soleil, toutes les valeurs de pixels d'une scène diminuent.

  • Si l'on multiplie l'intensité des pixels par $0.5$ (image deux fois plus sombre), la structure de l'objet reste parfaitement identifiable pour nous.
  • Pourtant, la distance pixel-à-pixel entre l'image claire et l'image sombre explose. Une voiture noire sous le soleil sera mathématiquement jugée plus proche d'une voiture blanche que de cette même voiture noire garée à l'ombre.

🔄 3. Les rotations et déformations géométriques

Dès qu'un objet pivote ou change d'angle de vue, la grille de pixels subit une redistribution totale. Le nez d'un visage ne s'aligne plus avec le nez du modèle de référence, mais avec une joue ou le front. La soustraction pixel-à-pixel compare des structures qui n'ont aucun rapport entre elles, détruisant toute possibilité de reconnaissance.

📉 Pourquoi les indices de texture (SSIM) ne suffisent pas

Face à ces limites, on pourrait être tenté d'utiliser des indices de comparaison d'images plus avancés, comme le SSIM (Structural Similarity Index Measure). Le SSIM évalue la similarité locale de luminance, de contraste et de structure sur de petites fenêtres de pixels.

Bien que le SSIM soit beaucoup plus robuste aux changements d'éclairage globaux que la simple distance L2, il reste fondamentalement une métrique de texture locale à bas niveau.

  • Le SSIM est incapable de faire de l'abstraction conceptuelle.
  • Il ne peut pas comprendre qu'une silhouette de camion rouge et une silhouette de citadine bleue partagent le concept sémantique de "véhicule". Pour le SSIM, les différences de couleurs locales et de contours l'emporteront toujours.

🧩 La solution : S'évader des pixels vers l'Espace Latent

Pour créer un moteur de recherche d'objets robuste et sans supervision, nous devons abandonner l'espace des pixels et projeter nos images dans un espace où les distances mesurent des concepts et non des couleurs.

C'est précisément le rôle de l'Espace Latent développé par des modèles d'apprentissage auto-supervisés comme I-JEPA ou Saccadic-JEPA :

  1. Au lieu de regarder les pixels individuels, on découpe l'image en patches et on les passe à travers un encodeur (un Vision Transformer - ViT).
  2. Les couches de Self-Attention de l'encodeur analysent comment chaque partie de l'image est liée aux autres, permettant de capturer la géométrie globale et le contexte.
  3. L'encodeur projette ces patches sous forme de vecteurs dans un espace à haute dimension (l'espace latent) où la proximité géométrique (similarité cosinus) traduit enfin une similarité sémantique.

Dans la partie suivante, nous verrons comment extraire et condenser cette information sémantique locale en un outil mathématique puissant : Le Vecteur de Concept ($q$).

🛠️ Défi de curiosité n°1

Imaginons une image en noir et blanc de dimensions $100\times100$ contenant un unique pixel blanc (valeur 255) situé en position $(50, 50)$, le reste de l'image étant totalement noir (valeur 0).

  1. Si on déplace ce pixel d'une seule case vers la droite pour le placer en $(51, 50)$, quelle est la distance euclidienne au carré ($L_2^2$) entre l'image originale et l'image modifiée ?
  2. Si on déplace maintenant ce pixel de 40 cases vers la droite pour le placer en $(90, 50)$, cette distance au carré change-t-elle ?
  3. Que concluez-vous sur la capacité des distances de pixels à mesurer la proximité de mouvement ou de déformation d'un objet ?
