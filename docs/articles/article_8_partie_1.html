Saccadic-JEPA — Partie 1 : Concept du Saccadic-JEPA

Apprendre des structures nettes à partir d'indices flous

Série Saccadic-JEPA : Vers une Vision Artificielle Biologiquement Inspirée Dans cette première partie de notre huitième article, nous présentons le paradigme Saccadic-JEPA. Nous allons voir comment cette architecture rompt la symétrie des modèles auto-supervisés classiques en opposant un Élève (Context Encoder) plongé dans le flou périphérique à un Professeur (Target Encoder) qui détient la vérité nette et détaillée.

🪞 Rappel : La symétrie de I-JEPA standard

Dans le modèle I-JEPA classique (Mono ou Multi-cibles), la tâche de pré-entraînement repose sur un masquage purement géométrique.

Les deux branches du réseau (le Context Encoder et le Target Encoder) travaillent sur la même image d'origine à résolution uniforme (par exemple, $32\times 32$ pixels) : 1. L'image d'origine est découpée en patches. 2. L'Élève (Context Encoder) reçoit uniquement les patches du contexte (ex: 48 patches visibles). 3. Le Professeur (Target Encoder) reçoit l'image entière pour en extraire les représentations cibles (ex: 16 patches masqués).

Bien qu'asymétrique dans son architecture (l'Élève a des gradients actifs, le Professeur est mis à jour par EMA), le modèle reste symétrique dans sa représentation d'entrée : les deux encodeurs analysent des patches issus de la même qualité d'image.

⚡ Le saut conceptuel de Saccadic-JEPA : L'asymétrie de résolution

Saccadic-JEPA rompt délibérément cette symétrie en introduisant une asymétrie de résolution inspirée de la vision humaine :

                               ┌───────────────────────────┐
                               │   Image Nette Originale   │
                               │        x (128x128)        │
                               └─────────────┬─────────────┘
                                             │
                      ┌──────────────────────┴──────────────────────┐
                      ▼                                             ▼
            [ Simulateur de Fovea ]                           (Pas de flou)
                      │                                             │
                      ▼                                             ▼
           ┌─────────────────────┐                       ┌─────────────────────┐
           │ Image Fovéée (Floue)│                       │ Image Nette Entière │
           │     foveated_x      │                       │          x          │
           └──────────┬──────────┘                       └──────────┬──────────┘
                      │                                             │
                      ▼                                             ▼
           ┌─────────────────────┐                       ┌─────────────────────┐
           │   Context Encoder   │                       │   Target Encoder    │
           │      (ÉLÈVE)        │                       │    (PROFESSEUR)     │
           └─────────────────────┘                       └─────────────────────┘

Dans ce paradigme : 1. L'Élève (Context Encoder - Branche $y$) : Reçoit une image fovéée (foveated_x), c'est-à-dire nette uniquement sur son point de fixation actuel $A$, et fortement floutée et désaturée sur toute sa périphérie. C'est la simulation de ce que l'œil humain perçoit. 2. Le Professeur (Target Encoder - Branche $x$) : Reçoit l'image originale nette (x), sans aucune dégradation. Il extrait les représentations latentes idéales à haute résolution.

🎯 Pourquoi cette asymétrie force-t-elle le modèle à être intelligent ?

En opposant une entrée floue à une cible nette, nous forçons le Predictor à accomplir une tâche beaucoup plus difficile et sémantique : il doit projeter des représentations périphériques dégradées pour reconstituer (imaginer) les caractéristiques géométriques précises et nettes de la cible.

  • Éviter le simple appariement de pixels : Si l'Élève voyait l'image nette, le modèle pourrait se contenter d'apprendre des corrélations de textures locales simples.
  • Apprendre les invariants de formes : Pour deviner qu'une tache floue et grise en périphérie correspond en réalité à la structure métallique d'une aile d'avion (qu'il doit prédire sous forme de caractéristiques nettes), le modèle est contraint de comprendre le concept d'un "avion" et de ses composants à un niveau d'abstraction élevé.

🛠️ Dans les entrailles du code : Le flux asymétrique

Cette asymétrie d'entrée est codée dans la méthode forward de notre classe SaccadicJEPA (jepa_lab/saccadic_models.py) :

def forward(self, x, context_indices, target_indices, context_fixations=None):
    # x : batch d'images originales nettes (B, 3, 128, 128)
    B = x.shape[0]

    # 1. Échantillonner le point de fixation A pour le contexte (Élève)
    if context_fixations is None:
        context_fixations = torch.rand(B, 2, device=x.device) * 1.6 - 0.8

    # 2. Générer l'image fovéée (floue en périphérie de A)
    foveated_x = foveate_batch(x, context_fixations)

    # 3. Le PROFESSEUR encode l'image originale nette
    with torch.no_grad():
        target_features = self.target_encoder(x) # Représentations nettes (B, N, D)
        idx_t = target_indices.unsqueeze(-1).expand(-1, -1, self.embed_dim)
        s_y = torch.gather(target_features, 1, idx_t) # Représentations cibles nettes

    # 4. L'ÉLÈVE encode le contexte fovéé (flou)
    s_x = self.context_encoder(foveated_x, context_indices) # (B, K_x, D)

    # ... le Predictor doit prédire s_y à partir de s_x ...

Remarquez la séparation nette à l'étape 3 et 4 : * self.target_encoder reçoit l'image brute nette x. * self.context_encoder reçoit l'image dégradée foveated_x.

🧠 Conclusion & Transition

En brisant la symétrie de résolution, Saccadic-JEPA émule le fonctionnement du cerveau : nous utilisons nos indices périphériques flous pour anticiper ce que nous allons regarder ensuite.

Mais quel est le rôle exact du Predictor dans cette configuration ? Comment parvient-il à faire le pont entre ces deux espaces de résolution différents pour projeter les représentations ?
